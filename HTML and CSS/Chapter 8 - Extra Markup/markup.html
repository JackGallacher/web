Doctypes

  • Becuase there have been several versions of html, each webpage should begin with a "DOCTYPE" declaration to a tell a broser which type of html the page is using e.g. < !DOCTYPE html >

Comments

  • If you want to add comments to your code that will not be visible in the users browsers, you can add the text between these characters: < !-- comments go here -- >

ID Attribute

  • Every HTML element can carry the "id" attribute. It is used to uniquely identify that element on the page. e.g. < h1 id = "page title">Title< / h1 >

Class Attribute

  • Every HTML element can also have a "class" attribute. Sometimes, rather than uniquely identifying one element within a document, you will want a way to identify several elements on the page. e.g. < p class = "important info" > important paragraph stuff < / p>

  • This also helps you add CSS to all class members giving them the same look without having to alter each one individually.

Block Elements

  • Some elements will always appear to start on a new line in the browser window. These are known as "block level" elements.

  • Examples of block elements are < h1 >, < p >, < ul > and < li >

Inline Elements

  • Some elements will always appear to continue on the same line as thier neighbouring elements. These are known as "inline" elements.

  • Examples of inline elements are < a >, < b >, < em > and < img >

Grouping Text and Elements in a block

  • The < div > element allows you to group a set of element together in one block level box.

  • For example, you might create a "div" element to contain all of the elements for the header of your site (the logo and the navigation), or you might create a "div" element to contain comments from visitors.

  • Adding an "id" tag to the div element allows you to modify a specific div section using css

  • To use the div tag, put your code between an opening < div id = "example" and closing < / div > tags.

IFrames

  • An iframe is like a little window that has been cut into your page - and in that page you can see another page. The term iframe in an abbreviation of "inline frame"

  • An iframe is created by using the < iframe > element. There are a few attributes that you will need to know to use it:

    • "src" The src attribute specifies the url of the page you want to show. e.g. < iframe src = "http//:www.bbc.co.uk" >< / iframe >

    • "height" and "width" are used to determine the size of the iframe. e.g. < iframe src = "http//:www.bbc.co.uk" height = "350" width = "450">< / iframe >

    • In HTML5, a new attribute called "seamless" can be applied where scrollbars (depreciated, used in HTML4) are not desired, using seamless doesnt require a vlue but can be given the value of "seamless". e.g. < iframe src = "http//:www.bbc.co.uk" height = "350" width = "450" seamless = "seamless">< / iframe >

Information about your pages

  • The < meta > element lives inside the head of a webpage and contains information about that webpage, it is usually joined by an name attrbute and a content attribute. e.g. < meta name = "description" content = "meta example" / >

  • Some defined values for this attribute that are commonly used are: "description", "keywords" and "robots". robots indicates if search engines should see this page, a values of "noindex" can be used if this page should not be added. A value of "nofollow" can be used if search engines should add this page but not any pages that it links to

  • The meta element also uses the "http-equiv" and "content" attribute in pairs. examples of "http-equiv" include "pragma" which prevents the browser from cacheing the page and "expires" which indicates when a page should expire.
