Forms...

  • A form is placed within the < form > tag

  • Every form element needs an "action" attribute. Its value is the URL for the page on the server that is receiving the form data. e.g. < form action = "http//:www.example.com/subscribe.php">

  • Forms can be sent using two methods, "get" or "post".

    • With the "get" method, values from the form are added to the end of the URL specified in the "action" attribute. this is ideal for short forms and when you are just retreiving data from a web server.

    • With the "post" method, values are sent in HTTP headers. You should use these when your form is; uploading a file, is very log, contains sensitive data or adds stuff to a database.

    • If no method is used, it will atomatically be set as the "get method"

    • e.g. < form action = "http//:www.example.com/subscribe.php" method = "get" > example below...

The Input tag

  • The < input > tag is used to create several different form controls. the value of the type attribute determines what kind of input they will be creating

  • The < type > attribute is used in the input tag to define what type of input will be eneted by the user. for example < input type = "text" >

Text Input

  • Text input is created by using the "text" input type. e.g.< input type = "text" >

  • A "name" attribute must be given to each form so the server knows which forms control each peice of data was entered into. e.g. < input type = "text" name = "username"/ >

  • The "size" attribute indicates the character width of the input box. although nowadays, this should be done with CSS. e.g.< input type = "text" name = "username" size = "15"/ >

  • The "maxlength" attribute is used to limit the amount of character that can be typed into the input box. e.g.< input type = "text" name = "username" size = "15" maxlength - "5"/ >

Text Area

  • The < textarea > element is used to create multi-line text input. Unlike other input elements, this is not an empty element. it should have an opening and closing tag.

  • Any text between the < textarea > and < / textarea > tags will appear in the text box when the page is loaded.

  • The "cols" and "rows" attributes should not really be used, the size of the text box should be controlled by CSS

  • As usual, a textarea tag goes within a forms tag.

Radio Buttons

  • Radio buttons are crated inside the input tags and given the type of "radio". e.g. < input type = "radio" / >

  • Radio buttons should be given a "name" attribute and "value" attribute. The "name" attribute is sent to the server and links to the specific radio buttons so data can be used. The "value" is the data value that is sent to the server. for example: < input type = radio name = "genre" value = "jazz" / >

  • You can also add a "checked" attribute to one radio button in your set which will load it checked. e.g. < input type = radio name = "genre" value = "jazz" checked = "checked"/ >

Checkbox

  • A checkbox is made the same way as a radio button, the only thing that changes is the "type" attribute. It changes to "checkbox". e.g. < input type = "checkbox" name = "genre" title = "Jazz" / >

  • With a checkbox, you can select multiple options, you can still select a box to be check too by using the checked = "checked" attribute

Drop Down List Box...

  • A drop down list (aka select box) allows on option to be selected from a drop down list. The < select > element is used to create this drop down box. it contains two or more < option > elements

  • Within the < select > element the attribute "name" must be given. This indicates the name of the form control being sent to the server, along with the value the user submitted. e.g. < select name = "devices">

  • The < option > element is used to specify the options that the user can select from. The words between the opening "option" tag and closing "option tags will be shown the user drop down."e.g. < select name = "devices">< option > iPod < / option >< / select >

  • The option element uses the "value" attribute to indicate the value that is sent to the server along with the name of the control if this option is selected. e.g. < select name = "devices">< option value = "iPod"> iPod < / option >< / select >

  • The "selected" attribute can be used to indicate the option that should be selected when the page loads using selected = "selected" e.g. < select name = "devices">< option value = "iPod" selected = "selected"> iPod < / option >< / select >

Multiple Select Box...

  • You can turn a drop down select box into a box that shows more than one option by adding the "size" attribute within the "select" element. Its value should be the number of options you want to show at once. e.g. < select name = "devices" size = 3>

  • You can allows users to select multiple options from this list by adding the "multiple" attribute with a value of "multiple" e.g. < select name = "devices" size = "3" multiple = "multiple" >

File Input Box...

  • If you want to allow users to upload a file, you will need to use a file input box.

  • To do this, add type = "file" into the "input" element. This type of input creates a box that looks like a text input followed by a browse button. e.g. < input type = "file" name = "user_file"/ >

  • To send data of your form to the server, add the type "submit" into a new input elemnet inside your form with the "value" attribute determining the words shown on the button. You can add a "name" attribute but it is not needed. e.g. < input type = "submit" name = "upload" value = "upload" >

  • When you are allowing users to upload files, the method attribute on the "form" element must have a value of "post". e.g. < form action = "http//:www.example.com/subscribe.php" method = "post" >
