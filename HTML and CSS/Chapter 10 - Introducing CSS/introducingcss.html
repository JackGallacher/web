Introducing CSS

Using external CSS

  • The < link > element can be used in a HTML document to tell the browser where to find the CSS file used to style the page. It is an emply element (meaning that it does not haveto have a closing tag) and it lives inside the < head > element. it should use three attributes

    1. "href": this specifies the path to the CSS file. (which is often places in a folder called CSS or Styles)

    2. "type": this attribute specifies the type of document being linked to. The value should be "text/css".

    3. "rel": thr specifies the relatinship between the HTML page and the file it is linked to. The value of this should be "stylesheet" when linking to a CSS file

  • An HTML page can use more that one CSS style sheet. To do this, it could have a < link > element for every CSS file it uses. some users use this method to have one CSS for fonts and another for layout

  • For example: < link href = "CSS/layout" type = "text/css" rel = "stylesheet"/ >

Using Internal CSS

  • You can also include CSS rules within an HTML page by placing them inside a < style > element, which usually sits within the < head > element of the page.

  • The style attribute should use the "type" attribute to indicate that the styles are specified in CSS. The value should be "text/css". e.g. < style type = "text/css" > CSS CODE HERE < / style >

How CSS rules cascade

  • If there are two or more rules that apply to the same element, it is important to understand which will take precedence.

  • If the two selectors are identical, the latter of the two will take precedence.

  • If one selector is more specific than the others, the more specific rule will take precedence over more general ones. for example, h1{} is more specific than *{}, p b{} is more specific than p, and p#inro is more specific than p

  • You can add "!important" after any property value to indicate that it should be considered more important than the other rules

Inheritance

  • If you specify the font-family or color properties on the < body > element, they will apply to most of the child elements. This is because the value of the font-family property is inherited by child elements.

  • You can force a lot of properties to inherit values from thier parent elements by using "inherit" for the value of the properties. e.g. "padding: inherit". for example, "background-color" and "border" properties are not inherited by a child class as the page would lokk messy if it it. but if you wanted them to, you would add "background-colour: inherit" to force it to be the same as its parent.
